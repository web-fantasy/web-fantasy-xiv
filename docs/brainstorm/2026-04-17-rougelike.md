# 肉鸽模式 GDD

> FF14 战斗体验 × 杀戮尖塔式爬塔肉鸽的同人小品游戏

---

## 1. 游戏概述

### 1.1 核心体验

玩家沿一条分支路径向上攀爬，途经小怪、精英怪、随机事件和奖励节点，最终挑战层boss。战斗系统复刻FF14的职业技能循环与boss机制处理，爬塔层提供每局不同的路径选择与装备构筑体验。支持断点续玩——玩家可以随时离开，下次从当前节点继续。

### 1.2 关键参数

| 参数         | 值                                                                       |
| ------------ | ------------------------------------------------------------------------ |
| 单轮目标时长 | 约 30 分钟（12 步 + Boss，见 7.3）                                       |
| 玩家等级上限 | 15                                                                       |
| 初始可选职业 | 3（剑术师 / 弓箭手 / 咒术师，需新建的基础职业）                          |
| 总职业数     | 9（3 基础新建 + 6 已实现进阶 job；玩家通过武器掉落从基础切换到进阶 job） |
| 视角/操作    | 沿用已有 FF14 boss战模拟器                                               |
| 存档         | 支持断点续玩，按节点保存进度                                             |

### 1.3 目标群体

定位：**FF14 友好 + 轻度扩圈**。默认面向 FF14 玩家，但通过可选教程与 UI 友好化，让被"FF14 × 杀戮尖塔"概念吸引的 roguelike 玩家也能上手。

核心原则：

- **不稀释核心体验**：FF14 式的演出、机制处理、术语（GCD / 能力技 / 魔晶石 / AOE 预告圈）全部保留，通过 tooltip 解释而非改名。
- **低技术下限、高技术上限**：以 **剑术师起步 → 武器切换到战士（生存型 DPS）** 为新手友好锚点，设计为"全程 111 平A 也能通关，但 DPS 明显低于正确循环"，给不想学的玩家留一条出路。**不能让整个游戏退化为平A**，其他职业和关卡机制必须保留 FF14 风味。
- **教程必须可跳过**：老玩家一键略过，新玩家从零递进学习。

> 设计依据：MVP 试玩反馈中，未接触 FF14 的朋友普遍反馈"游戏太难、技能说明复杂、懒得学习"。剑术师的 111 兜底是给这部分玩家的入场券，而非全局简化。

### 1.4 教程与新手引导

基于现有单关模拟器的教学能力改造，形成一条**可跳过的线性教程**：

| 步骤 | 教学内容                                                |
| ---- | ------------------------------------------------------- |
| 1    | 移动控制                                                |
| 2    | 普通攻击（区分近战/远程职业的作用距离）                 |
| 3    | 强化技能（及时使用、冷却管理）                          |
| 4    | 治疗自愈（HP 恢复时机）                                 |
| 5    | 防御/机动（减伤技能、突进技能用于躲避来不及走位的机制） |

教程入口：

- **首次启动**：弹出"是否学习游戏基础？"的提示，默认推荐新玩家进入，老玩家一键跳过。
- **主菜单常驻入口**：任何时候都可以回去重学某一节。
- **不强制**：直接开局是允许的，但新手选剑术师时，首节小怪战内会显示 contextual 提示（可关闭）。

**实现方式**：教程塔是一个**固定路径的手工塔**（hand-crafted tower，见 7.4），线性流程，不走随机生成。

## 2. 局内系统（爬塔肉鸽）

### 2.1 塔层结构

每局游戏是一座塔，塔由若干层组成，每层包含多个节点，节点之间形成分支路径（类似杀戮尖塔的地图）。

```
        [BOSS]              ← 层 Boss
       /      \\
     [?]      [E]           ← 随机事件 / 精英怪
    / | \\    / | \\
  [M] [R] [M] [M] [?]      ← 小怪 / 奖励 / 事件
   |   |   |   |   |
  [M] [?] [E] [R] [M]
   \\   |   |   |  /
    \\  |   |  | /
     [START]                ← 起点
```

玩家在每个节点完成后，从可达的下层节点中选择下一步走向。路径选择是核心策略：是稳扎稳打刷小怪攒水晶，还是冒险挑战精英怪博更好的掉落。

### 2.2 节点类型

| 节点      | 图标 | 内容                                                                             |
| --------- | ---- | -------------------------------------------------------------------------------- |
| 小怪战斗  | ⚔️   | 较弱的敌人，击杀获得水晶。低风险低回报                                           |
| 精英战斗  | 💀   | 较强的敌人，有独特机制。击杀必定掉落 3 选 1 奖励（见 2.7）                       |
| Boss 战斗 | 👑   | 每座塔的最终关卡（见 2.13）                                                      |
| 篝火      | 🔥   | 安全节点，可升级/配装/奉献；Boss 前的最后一步必定为篝火（参考杀戮尖塔；见 2.12） |
| 奖励      | 🎁   | 免费获得水晶、魔晶石或其他资源；单层塔的中点必定为奖励层（参考杀戮尖塔）         |
| 随机事件  | ❓   | 随机触发特殊事件（见 2.3）                                                       |

#### 2.2.1 第 0 节点（每层起点）

参考杀戮尖塔，每层塔的起点是一个固定的**第 0 节点**（MVP 为 1 层，即单次游戏仅 1 个第 0 节点）。必定提供：

- **1 件随机武器**：即刻切换到对应进阶 job（见 2.11），给玩家起步 build 方向
- **3 选 1 策略卡**：从策略卡池抽 3 张，玩家选 1 张（见 2.17）

第 0 节点不是可选分支，而是路径的固定起点——玩家在此获得初始补给，然后从第 1 步开始进入分叉。

> 设计意图：原 GDD 基础职业只有 1-3 个技能，前 1-3 步战斗可能单薄。第 0 节点的随机武器让玩家第一场战斗就能进入"进阶 job"体验；策略卡提供额外构筑维度，呼应杀戮尖塔"遗物"的长效被动。

### 2.3 随机事件

随机事件节点会触发一个随机事件，可能包括：

- 发现一个宝箱，但打开需要付出代价（消耗水晶/HP）
- 遇到一个 NPC，提供交易（用魔晶石换魔晶石、用水晶换武器、用水晶提升魔晶石的品质等）
- 触发 buff/debuff（持续若干节点）
- 遭遇陷阱（战斗前受到伤害/debuff）
- 遇到治愈绿洲：恢复 1 点决心（见 2.14）
- 其他惊喜或危机

> 具体事件池 TODO，核心原则：每个事件都应该包含一个有意义的选择。

### 2.4 侦察与逃跑

- 点击任意节点时，均有一个确认/取消弹窗：
  - 显示节点基本信息（已侦察的敌人情报、奖励预览等）
  - 战斗节点有额外的**侦查**选项
  - 显示本场战斗激活的**场地机制**及其触发条件（如超越之力的当前状态，见 2.14.3）
- 战斗节点（小怪/精英）在路径地图上只显示节点类型，**不显示具体敌人情报**。
- 每个战斗节点**首次**侦察需要付出 **1 水晶**，下一次点开将免费查看已解锁的情报。
- 也就是说玩家可以付出大量代价，侦查接下来的每一个可选战斗分支，然后确定攻略其中一个；也可以直接闭眼选择进入。
- 一旦确认选择，则锁定路线（战斗失败出来也不能重新选了，方案既定）。
- **不能跳过战斗**：进入战斗节点后必须至少尝试一次才会出现"放弃"按钮（见 2.14）。侦察让玩家判断"值不值得试"，但不能免战。

> 设计意图：侦察+逃跑机制继承了原版”走近boss才能看到信息”的不确定性，同时在爬塔框架下更加结构化。“侦察后逃跑有代价”保证了路径选择本身的重量感。

### 2.5 战斗系统

沿用已有的 FF14 boss战模拟器，包括：

- 职业技能循环（GCD / 能力技）
- boss / 精英怪技能表演与机制处理（AOE、分摊、击退等）
- 状态效果系统
- 简单的敌人 AI

| 战斗类型 | TTK     | 复杂度                                         |
| -------- | ------- | ---------------------------------------------- |
| 小怪战   | ~90s    | 轻机制 + 主题差异（详见 2.5.1）                |
| 精英战   | 2-3 min | 独特机制，可能是怪异的机制马拉松（2.5.2 待定） |
| Boss 战  | 3-5 min | 完整 FF14 式遭遇战，多阶段（见 2.13）          |

具体战斗细节参见模拟器文档（TODO: 链接）。

#### 2.5.1 小怪战设计原则

玩家一局会打 ~6-7 场小怪战，设计目标是**在短时间内保持新鲜感**，既不变成纯沙袋 DPS 测试，又不至于负担过重。

- **强度档位**：S2 轻机制——每只小怪含 1-2 个需躲避 AOE，无多阶段切换
- **主题差异**：不同小怪有明确主题（冰/火/多段攻击/连续点名/追踪圈等），机制虽简单但需要即时识别并反应
- **TTK 目标**：~90s，允许玩家打完 1-2 个完整爆发循环，验证 build 输出能力
- **软性超时（不做狂暴秒杀）**：90s 之后小怪进入"超时"状态——之前展示过的机制开始**循环重复**，并叠加 1 层"愤怒"buff（小怪造成伤害提升）；每再过 90s 额外叠 1 层。玩家不会被瞬间 wipe，但失血加速会逼迫其更高效收尾。绝不做到时直接斩杀的狂暴机制
- **下限保证**：符合 1.3 低技术下限原则——剑术师 111 平 A 也能勉强通关，只是慢（会被愤怒 buff 拖久一点，但不会被强行打死）

> 小怪战是"验证你 build 能不能 push DPS"的练手场，也是路径上的相对轻度节点。每场都压力极大会让玩家到 Boss 心力交瘁；每场都是沙袋又劝退。软性超时既保持了"高手能打快"的奖励，也留了"菜菜慢慢磨"的后路。

#### 2.5.2 精英战设计原则

精英战是"高风险高回报"节点（失败 -1 决心，胜利必掉 3 选 1），设计原则是**每次精英都有惊喜**——GDD 所说"怪异的机制马拉松"不是口号，而是关键差异化。

**强度档位**：E4 混合池——MVP 做 3 个不同类型的精英进池，随机抽取。

**类型样板**（每类 MVP 至少做 1 个）：

| 类型             | 设计重点                                                     |
| ---------------- | ------------------------------------------------------------ |
| **DPS Check 向** | 机制偏简单，但必须在限时窗口内打出足够伤害，否则进入愤怒循环 |
| **AOE 马拉松向** | 输出压力低（甚至可以不打），但 AOE 连环密集，连吃 3-4 下必死 |
| **小游戏向**     | 插入非战斗小游戏段落，完成情况影响后续 DPS check 的宽松度    |

**TTK 目标**：120-180s。同样采用**软性超时**（机制循环 + 叠愤怒 buff），不做狂暴秒杀。

**强度对齐**：以"FF14 挑战模式 boss 第一阶段"为基线——核心玩家能单次通关，新手可能需要 1-2 次熟悉。

**FF14 参考关卡灵感库**（类型 → 关卡 → 核心机制；持续扩容，为后续扩池储备）：

- **AOE 马拉松向 → 巴儿达木霸道**：背对 / 远离 / 放置陨石后躲背面；答错叠 debuff，4 层即死
- **小游戏向 → 朱雀歼殛战跳舞机**：限时按 WASD 组合键；跳舞机结束后 boss 现身，buff 层数决定后续 DPS check 宽松度（需 4 层以上才有机会过）
- **小游戏向 → 伊瓦莉斯质数机器人**：玩家血量被强制设为 1-4，场地有编号 1-4 的圆圈（踩上将血量改为对应值）；boss 读条"质数" / "5 的倍数" 等数学条件，站错即死

> 精英战是本游戏"FF14 演出设计的高光场"，也是玩家反复游玩的动力——"这次又抽到什么怪？"。MVP 做 3 个是底线，后续扩池优先级最高。

### 2.6 等级与水晶

- 击杀敌人获得**水晶**。
- 在**篝火**处消耗水晶进行升级，提升基础属性。
- 等级上限：**15 级**。
- 小怪战斗掉落少量水晶，精英战斗掉落较多水晶。

### 2.7 精英/Boss 掉落（3 选 1）

击杀精英怪或 boss 后必定掉落以下 3 件物品，玩家**选择其中 1 件**获取：

1. **1 把随机武器**
2. **1 颗随机魔晶石（A）**
3. **1 颗随机魔晶石（B）**

这意味着每次击杀精英/boss，玩家要在”切换职业/强化 build”之间做出抉择。

### 2.8 武器

| 属性        | 说明                                                               |
| ----------- | ------------------------------------------------------------------ |
| 职业绑定    | 切换武器 = 切换到对应 **进阶 job**（基础职业 → 进阶 job，见 2.11） |
| 固定词条 ×2 | 必定包含该职能的核心属性补正（如法杖 = 技速 + 魔法攻击）           |
| 随机词条 ×1 | 从全局词条池中随机                                                 |

武器决定当前职业，因此选择武器意味着可能需要重新调整魔晶石配装。

### 2.9 魔晶石

| 属性     | 说明                                                               |
| -------- | ------------------------------------------------------------------ |
| 词条数量 | 随机 1–3 条                                                        |
| 词条类型 | 随机（见词条体系）                                                 |
| 词条等级 | 随机                                                               |
| 携带上限 | 无限制                                                             |
| 激活上限 | 愿景：min(当前等级, 15)，最低 3；**MVP 简化为固定 5 槽**（见 7.1） |

玩家可以自由选择激活哪些魔晶石。核心策略：在有限的激活槽位内，搭配出最优的词条组合。

### 2.10 词条体系（简化版）

采用简化的属性系统，分为以下大类：

| 词条类型 | 效果                        | 适用倾向      |
| -------- | --------------------------- | ------------- |
| 物理攻击 | 提升物理技能伤害            | 近战 / 远物理 |
| 魔法攻击 | 提升魔法技能伤害            | 法系          |
| 防御     | 减少受到的伤害              | 通用 / 坦向   |
| 技速     | 缩短 GCD / 同比缩短咏唱时间 | 通用          |
| 暴击率   | 提升暴击概率                | 通用          |
| 暴击伤害 | 提升暴击时的额外伤害        | 通用          |
| 生命值   | 提升最大 HP                 | 通用 / 容错向 |

> 设计意图：词条类型足够少，让玩家能快速判断价值；但通过数值等级的随机性，仍然保持 build 多样性。不同职能有属性补正差异，但只要魔晶石搭配合理，任何职业都可以打出高伤害。

### 2.11 职业系统

职业分两层：**3 个基础职业**（开局可选，需新建）+ **6 个进阶 job**（已实现，丰富循环）。玩家从基础职业起步，通过拾取武器切换到进阶 job。

#### 开局可选职业（基础，新建）

技能 1-3 个，循环简单，作为"低门槛起步"。呼应 1.3 目标群体 A2 的新手友好原则。

| 职业   | 职能     | 特色               |
| ------ | -------- | ------------------ |
| 剑术师 | 物理近战 | 攻防均衡，容错率高 |
| 弓箭手 | 物理远程 | 持续输出，机动性强 |
| 咒术师 | 魔法远程 | 爆发伤害，需要读条 |

#### 进阶 Job（已实现 6 个）

拾取对应武器后切换。每把武器对应 1 个进阶 job。

| 进阶 Job | 定位         | 特色                                                      |
| -------- | ------------ | --------------------------------------------------------- |
| 战士     | 生存型 DPS   | DPS 略低但自愈手段多，容错高；111 平 A 也能通关（见 1.3） |
| 圣骑士   | 生存型 DPS   | 减伤工具丰富，适合机制型精英战                            |
| 暗黑骑士 | 生存型 DPS   | 血换爆发，高风险高回报                                    |
| 武士     | 近战爆发 DPS | 高爆发、蓄力循环                                          |
| 吟游诗人 | 远程持续 DPS | 持续输出，机动性强                                        |
| 黑魔法师 | 远程爆发 DPS | 爆发伤害最高，需要读条                                    |

> 注：战士/圣骑士/暗黑骑士虽然在 FF14 本体是 tank role，但在本游戏单人爬塔语境下退化为"DPS 略低 + 自愈/减伤手段多"的生存型定位，适合新手入门，不承担传统 MMO 的嘲讽/团队保护职责。

> 注：进阶 job 说明为初稿，需后续逐个核对现有实现校准描述。**武器与基础职业无强制映射**——任何基础职业拾取任意武器都可切换到对应进阶 job（例：弓箭手拾取剑盾即可切为圣骑士；咒术师拾取大剑即可切为暗黑骑士）。这种设计鼓励玩家敢于尝试不同 build 路线。

#### 武器切换职业

通过 **第 0 节点**（见 2.2.1）、**精英/Boss 掉落**（3 选 1）、**随机事件** 等途径获得武器。每把武器对应上表中的 1 个进阶 job；切换武器 = 切换 job；武器也可能是同 job 的更好版本。

> 职业设计原则：基础职业是"低门槛上手"，进阶 job 是"搭 build 跑完整循环"。每个 job 要么擅长爆发输出，要么擅长极致容错。

### 2.12 篝火

篝火是路径上的安全节点，功能包括：

- **升级**：消耗水晶提升等级。
- **配装**：调整魔晶石激活组合、切换武器。
- **奉献**：献祭魔晶石以换取 +1 决心（见 2.14 决心）。

### 2.13 Boss 战

每座塔的路径最终汇聚到一个 boss 节点：

- Boss 是一场完整的 FF14 式遭遇战，**2-3 个阶段**，TTK **3-5 min**。
- 使用此前收集的全部装备（武器 + 激活的魔晶石）进行攻略。
- **硬狂暴机制**：与小怪/精英的软超时不同，Boss 有 **硬狂暴秒杀**——超时未击杀即全场 wipe。作为终局高光，Boss 应具备 FF14 团本级的压力感；失败 -2 决心已鼓励谨慎挑战，硬狂暴保留终局紧张感
- **超越之力翻盘 buff**：进入 Boss 战时若决心 ≤ 2（最后一次尝试），自动获得全属性加成（见 2.14.3）
- **死亡机制**：Boss 战中死亡扣除 **2 点决心**（见 2.14），HP 回满后再次挑战。
- 决心耗尽（≤ 0）则本局结束，仍然会结算奖励（但金币大幅减少）。

### 2.14 决心（Determination）

局外生命池，独立于战斗内 HP，作为"玩家在本局的容错资源"。命名致敬 Undertale 的 Determination 与 FF14 的同名属性。

| 参数                         | 值               | 备注                                              |
| ---------------------------- | ---------------- | ------------------------------------------------- |
| 初始决心                     | 5 ❤️             | 开局给定                                          |
| 决心归零                     | 本局结束         | 进入结算（按当前收集情况）                        |
| 小怪/精英失败（选重试）      | -1 ❤️            | 扣减时机 = 战斗结束弹窗弹出瞬间；HP 回满      |
| 小怪/精英失败（选放弃）      | -1 ❤️            | 扣减时机 = 战斗结束弹窗弹出瞬间，重试/放弃同档；节点标记"已通过"，获得放弃低保（见 2.14.1）|
| 放弃选项出现时机             | **仅战斗失败后** | 进入战斗前**无**放弃按钮；玩家必须至少尝试一次    |
| Boss 失败                    | -2 ❤️            | 扣减时机同上；HP 回满后再次挑战；决心 ≤ 0 时直接结算；放弃 = 整局结束（不走低保）|
| 篝火奉献（魔晶石 → +1 决心） | +1 ❤️            | 见 2.12                                           |
| 决心上限                     | 5（MVP）         | 局外养成可永久扩容（未来版本）                    |

> **Phase 5 更新（2026-04-18）**：决心扣减时机统一到"战斗结束弹窗弹出瞬间"，不再区分重试 / 放弃按钮档位（重试与放弃同档扣减）。放弃仍提供 50% 水晶低保（小怪/精英）或直接进结算（boss 独有：放弃 = 整局结束）。

**战斗内 HP 策略**：每场战斗开始 HP 回满（纯 FF14 式），治疗魔晶石/技能只在本场战斗内有效。决心只在"战斗失败"时扣除，不与战斗 HP 混淆。

> **沉没成本陷阱设计**：放弃选项**只在战斗失败后**出现——玩家可能不信邪在同一节点死 2-3 次才甘心放弃，每次失败都要做"再送一次还是收手拿低保"的心理抉择。这是 roguelike 特有的张力，比"进战斗前就能 skip"更有重量。

#### 2.14.1 放弃低保

战斗失败后选择放弃的节点，按该节点正常奖励的 **~50%** 给出（水晶 / 3 选 1 掉落酌减），给玩家"打不过就舍一半奖励走人"的退路，同时避免全送头到 Boss 的 degenerate 玩法。

**目标经验曲线**（MVP 数值调优基线）：

| 玩家行为                   | Boss 前预计等级 | 水晶结余                 |
| -------------------------- | --------------- | ------------------------ |
| 全胜，正常挑战             | 13 级           | 刚好用完                 |
| 全胜 + 大胆挑战所有精英    | 15 级（满级）   | 溢出，可干更多事         |
| 全部放弃（仅走非战斗节点） | 8-9 级          | 刚够喘气（黑夜君临风味） |

#### 2.14.2 决心恢复途径

- **随机事件**：特定事件可提供 +1 决心（见 2.3）
- **篝火奉献**：魔晶石换 1 决心（见 2.12）
- **局外养成**：可永久提升决心上限（MVP 不做）
- **深层迷宫负向词条**：进入高难度迷宫时可能携带"开局 -1 决心"的 debuff 词条（MVP 不做）

#### 2.14.3 超越之力（The Echo）

当玩家进入 **Boss 战**，且 **决心 ≤ 2**（即这场失败本局就结束）时，自动获得 **超越之力** buff：

| 属性         | 值                                            |
| ------------ | --------------------------------------------- |
| 触发条件     | 进入 Boss 战 时决心 ≤ 2（最后一次尝试）       |
| 效果         | 全属性加成 **+25%**（初始数值，调优阶段可改） |
| 作用范围     | 仅 Boss 战内有效；通关/死亡后立即失效         |
| 深层迷宫模式 | **无效**（高难度玩家不享受翻盘 buff；见 7.2） |

**实现方式（场地机制）**：
"超越之力"是一类**通用场地机制** (battlefield condition)——"本场战斗在决心 < X 时给予 buff Y"。MVP 只把这个机制配置给 Boss 战（阈值 2、buff = 全属性 +25%），其他战斗不挂载。未来可复用到特殊事件战斗、Boss 第二阶段、深层迷宫专属遭遇等场景，只需改配置。

**UX 提示**：进入战斗前的确认弹窗（见 2.4）必须**显示该战斗激活了哪些场地机制**，包括超越之力当前是否满足触发条件（"你的决心 = 2，此战会获得 +25% 全属性"）。让玩家决策时知情。

**设计意图**：给新手 / 翻车局的玩家一次"背水一战"翻盘机会，契合 1.3 目标群体 A2 "低技术下限"原则。加成只让"本来差一口气"的玩家有希望过关，并不保证胜利——完全没练 build 的玩家照样会输。

**为什么只限 Boss 战（不扩展到所有"失败即 GG"的战斗）？**
防止"鸡爪流"degenerate 玩法——若小怪/精英战也能触发 buff，玩家可能故意开局送血到 1 决心，后续每场小怪都享受永久 +25%。Boss-only 天然关闭这个漏洞：送血到 1 决心后，中途任何小怪/精英失败即 GG，玩家不敢赌。也匹配 FF14 本体"超越之力（The Echo）"的使用场景——只在副本 Boss 战中生效，小怪战不会触发。

> **命名来源**：直接沿用 FF14 本体的"超越之力（The Echo）"概念——原游戏里，光之战士受到光之加护后拥有的能力；在旧版本副本的机制里，反复挑战失败会自动获得该 buff，按失败次数 stack 提升属性。本游戏"决心 ≤ 2 时自动激活 +25% 全属性"完全对应这个"反复失败后 scaling up"的精神与 lore。

> 注意：此处的"超越之力"与 FF14 另一个机制"极限技 / Limit Break"无关——后者是 4+ 人队伍战斗中积攒的共享大招，与本单人翻盘 buff 是两回事，易混淆故提示。

### 2.15 死亡惩罚

| 场景                        | 惩罚                                            |
| --------------------------- | ----------------------------------------------- |
| 小怪/精英战斗中死亡（重试） | 扣 1 决心，从当前节点重试（HP 回满）            |
| 小怪/精英战斗中死亡（放弃） | 不扣决心，获得放弃低保（见 2.14.1），标记已通过 |
| Boss 战中死亡               | 扣 2 决心，重新挑战（HP 回满）                  |
| 决心耗尽                    | 本局结束，按当前收集情况结算（金币大幅减少）    |

### 2.16 结算

| 结算因素       | 影响                         |
| -------------- | ---------------------------- |
| 魔晶石收集品质 | 收集到的魔晶石越好，金币越多 |
| Boss 击杀用时  | 用时越短，金币越多           |
| 是否通关       | 通关有大量额外金币           |

结算产出：**金币**（用于局外甲胄抽卡）。

### 2.17 策略卡

类杀戮尖塔"遗物"的概念：**局内持久生效的被动效果卡**，为 build 提供长期方向。

- **来源**：第 0 节点必定 3 选 1（见 2.2.1）；后续随机事件也可能提供（TBD）
- **持有上限**：MVP 无上限
- **生效范围**：仅本局内，局外不保留
- **类型举例**（MVP 卡池待设计）：
  - **数值类**："暴击率 +10%"、"全伤害 +5%"
  - **规则类**："每 3 步恢复 1 决心"、"侦察免费"、"篝火升级打折"
  - **反面卡**（高风险高回报）："全属性 +30%，但每走一步 -1 水晶"

> MVP 策略卡池规模 **15-20 张**。核心原则：卡片效果应**长期影响 build 思路**，而非一次性增益。一次性增益留给魔晶石和战斗内 buff。

## 3. 局外系统

### 3.1 甲胄（永久装备）

甲胄是局外获取的永久装备，跨局生效。

| 属性           | 说明                                          |
| -------------- | --------------------------------------------- |
| 部位           | 帽子 / 衣服 / 腰带 / 裤子 / 鞋（共 5 个部位） |
| 词条数量       | 随机 1–3 条                                   |
| 词条类型与等级 | 随机                                          |
| 穿戴规则       | 每个部位最多穿戴 1 件                         |
| 词条上限       | 5 部位 × 最多 3 词条 = 最多 15 条永久加成     |

#### 3.1.1 数值平衡策略

甲胄**不做硬性数值封顶**——玩家可以通过长期积累把词条堆到理论上限 15 条。平衡通过以下机制消化：

- **加成天然为加算**：依据 `docs/job-balance.md`，项目内所有属性增益采用 **加算**（`1 + Σ bonus`），只有减伤是乘算。因此 15 条词条总加成线性增长，不会指数爆炸——"看起来很多"但实际收益温和，`special` 伤害还穿透所有防御
- **深层迷宫动态难度**（2.X post-MVP）：难度 +1 / +2 分档，敌人数值 + 机制同步加强；某些层次开局强制塞负面策略卡
- **绝境战硬核选项**（见 3.3）：允许玩家选"不穿甲胄入场"换取更高奖励

> 这样"穿甲越来越强"的爽感被保留，不至于在高难内容中变成"穿满就无脑通"。

### 3.2 甲胄商店（抽卡）

- 消耗**金币**进行抽卡，获得随机甲胄。
- 可使用**定轨券**保底：指定 3 个词条类型，但每个词条的数值（效果量）仍然随机。
- 可额外消耗定轨券，使其中 1 个词条**必定为最高稀有度**。

### 3.3 绝境战

独立于爬塔主玩法的高难度模式。

| 属性     | 说明                                                                                     |
| -------- | ---------------------------------------------------------------------------------------- |
| 形式     | 无存档点的高难度遭遇战                                                                   |
| 职业选择 | 自由选择任意已解锁职业（武器加成固定）                                                   |
| 甲胄入场 | **可选**——穿：常规奖励；不穿（硬核挑战）：奖励 **+50%** 定轨券（初始数值，调优阶段可改） |
| 奖励     | 定轨券（见 3.2）                                                                         |

#### 3.3.1 甲胄入场策略

允许玩家在入场时选"穿甲胄"或"不穿甲胄"，作为**难度选择开关**：

- **穿甲胄**（常规路径）：让养成有回报，"甲胄抽卡 → 绝境战 → 定轨券 → 更好甲胄"的闭环成立
- **不穿甲胄**（硬核路径）：去掉永久加成，纯靠技巧 + 职业理解击杀；+50% 定轨券作为硬核钩子

> 设计意图：绝境战本身构筑维度较少（固定武器、无魔晶石、无策略卡），禁穿甲胄会让体验变单薄。允许自选"不穿"作为硬核挑战，服务 hardcore 群体并提供额外留存钩子。日后深层绝境战梯度（如"绝境战+2"）可固定强制禁甲胄作为最高难度。

## 6. 设计备忘（讨论记录）

以下是设计过程中讨论过的关键决策与理由，供后续参考：

**为什么从大地图+毒圈改为爬塔？**
大地图+毒圈的实现成本高（大地图渲染、毒圈逻辑、移动系统），且与FF14的”站桩式”战斗体验存在节奏冲突。爬塔路径选择保留了”探索+决策”的核心，砍掉了最重的实现成本，同时天然支持断点续玩。

**为什么保留侦察+逃跑机制？**
继承原版”走近boss才能看到信息”的不确定性。在爬塔框架下，表现为”进入战斗节点后可以先看敌人信息再决定是否逃跑”。逃跑有少量代价（丢失水晶），保证路径选择的重量感。

**为什么 3 选 1 而不是分开掉落？**
强制取舍。每次击杀精英/boss 都要在”换职业可能性”和”强化当前 build”之间做决策，增加策略深度。

**为什么甲胄是抽卡而不是明码标价？**
配合定轨券形成”爬塔主线产出金币 → 绝境战产出定轨券 → 两者共同作用于甲胄获取”的双轨激励，让两种玩法都有存在意义。

**原版大地图设计的遗产**
原版的分阶段区域设计（外圈低级→内圈高级、debuff式毒圈限制发育时间）在爬塔版中自然映射为”越往上节点越难”的结构。毒圈的”限制发育时间”功能由有限的路径长度（节点数）替代。

**为什么目标群体定位在 A2（FF14 友好 + 轻度扩圈），而不是彻底简化？**
MVP 试玩反馈显示 FF14 外的玩家确实卡在"技能多、说明难"上，但彻底简化成平A会失去 FF14 演出设计的灵魂——这是本项目的差异化卖点。折中方案是"职业分难度曲线"：剑术师提供 111 通关路径，其他职业保留完整循环。降低门槛与保留深度并不矛盾，关键是让玩家能自选。

**为什么把爬塔生命池叫"决心"？**
双重致敬：① Undertale —— Determination 是主角在失败中反复重来的核心力量，完美契合 roguelike "再来一次" 的精神；② FF14 —— Determination（信念）本就是 FF14 的一个属性词条，核心受众会立刻 get 到。"决心"也比"挑战心数/塔心"更不容易和 3.x 局外系统的"局外"语义混淆。

**为什么"放弃"只在战斗失败后才出现？**
三个原因：① 防止玩家看一眼就 skip 的 degenerate 玩法，保证路径推进的沉重感；② 形成**沉没成本陷阱**——玩家不信邪在同节点死 2 次后已经 -2 决心，会纠结"再送一次还是收手"，这是 roguelike 的心理张力来源；③ 与侦察机制互补：1 水晶的侦察让玩家判断"值不值得试"，但**免战**要付出至少 1 次失败的代价（-1 决心 + 低保奖励），保证侦察决策仍有重量。

  - 新建 **3 个基础职业**（剑术师 / 弓箭手 / 咒术师）——技能 1-3 个，循环简单，作为开局起步

### 7.3 塔结构参数

**1 层塔，12 步深度**（从起点到 Boss 前经过 12 个节点，单局预计约 30 min）：

> 术语说明：GDD 中的"层"指塔的大区段（每一层有独立 Boss）。本节的"步/节点"是**一层内**的单次交互节点。MVP 只做 1 层 × 12 步。

- **玩家路径长度**：12 步 + 1 Boss
- **每步分叉数 K**：固定层 K=1（所有路径强制汇合），其他层默认 K=2，关键决策位 K=3

**固定层**（参考杀戮尖塔，所有分支强制汇合）：

| 节点位置 | 节点类型 | 备注                            |
| -------- | -------- | ------------------------------- |
| 第 6 步  | 奖励 🎁  | 单局中点，所有路径汇合领取奖励  |
| 第 12 步 | 篝火 🔥  | Boss 前最后一步，必定为休息节点 |
| 终点     | Boss 👑  | 层 Boss，完整 FF14 式遭遇战     |

**非固定层**（第 1-5, 7-11 步，共 10 步）节点类型初始权重：

| 节点类型 | 权重 | 备注                     |
| -------- | ---- | ------------------------ |
| 小怪战   | 50%  | 主干节点                 |
| 精英战   | 15%  | 不宜过多                 |
| 随机事件 | 15%  | 增加探索感               |
| 奖励     | 10%  | 额外奖励机会             |
| 篝火     | 10%  | 额外补给点，用于路径策略 |

**生成约束**（硬性规则，防止不合理分布）：

- 第 1 步必为小怪战（教程衔接、新手友好）
- 精英战不连续两步出现
- 篝火不与第 12 步相邻
- 奖励不与第 6 步相邻
- 每条路径（从起点到 Boss）至少包含 1 次精英战

> 初始数值与约束以配置化形式存在（见 7.4 数据与引擎分离原则），调优阶段可迭代。

### 7.4 技术架构原则

- **数据与引擎分离**：词条池、魔晶石词条出现概率与等级分布、3 选 1 掉落权重、节点类型分布、敌人参数等**全部以外部配置（YAML/JSON）存在**。代码只负责消费数据，设计师独立调参不动代码
- **场地机制作为可复用模式**：战斗级别的条件规则（如超越之力"决心 < X → 给予 buff Y"）抽象为通用 battlefield condition 配置项。每场战斗可声明零到多个场地机制。MVP 只挂载到 Boss 战，但架构上支持任意战斗挂载，便于未来扩展
- **沿用现有分层**：core / entity / combat / skill / timeline / ai / arena 不变，新增"爬塔元层"管理节点状态、路径、局内持久化
- **持久化使用 IndexedDB**：玩家持久数据（当前塔运行、甲胄收藏、统计、成就）存 IndexedDB，封装库选用 **localspace**（localforage 的现代替代品，去掉无用 driver，API 几乎一致）。localStorage 仅用于轻量 UI 偏好（如主题、当前基础职业选择）。理由：IndexedDB 配额更大、异步不阻塞主线程、支持结构化数据
- **Seeded PRNG**：所有随机生成（塔图、魔晶石词条、敌人池抽取、事件选择、策略卡抽取等）使用 **seeded 伪随机**，不使用 `Math.random`。塔运行 state 保存 seed，支持：
  - 断点续玩（重现相同随机结果）
  - 种子赛 / 每日种子（post-MVP 弱社交钩子）
  - QA 复现 bug
- **塔图支持两种源**：
  - **Random**：按 7.3 参数 + seed 生成（主路径）
  - **Hand-crafted**：从 YAML 加载固定图（用于 1.4 教程塔、post-MVP 剧情塔、QA 测试场景）
  - 两种源都实现相同的 `TowerGraph` 类型，下游代码无需区分
- **Tower mode 单路由 / URL 不承载游戏状态**：tower 运行期间玩家始终停留在 `/tower`；内部通过 `TowerStore.phase` 切换子视图（menu / select-job / in-path / in-combat / ended）。战斗不 navigate 到 `/encounter/[id]`，而是在 `/tower` 内嵌 `<EncounterRunner>` 组件，encounter 配置从 store 读取。这样 URL 无 id 可改、无结算页可伪造、浏览器 back 也不会导致状态错乱
- **敏感路由防御式校验**：
  - `/encounter/[id]` 仅服务 standalone simulator，忽略任何 `?tower=*` 查询参数（丢弃并 console.warn）
  - 如未来确需 tower-aware 入口参数，必须校验 `TowerStore.phase === 'in-combat'` + URL id 与 store 当前 encounter id 一致 + runId 与持久化 run 一致；任一不匹配即 `router.replace('/tower')`
  - 技术承认：DevTools 直接改 Pinia store 的作弊不在客户端防御范围
- 更细粒度的模块划分、组件拆分、IndexedDB schema 在后续 implementation plan 敲定（见 superpowers:writing-plans）

### 7.5 Post-MVP Roadmap（已记录，暂不展开）

MVP 打通后按优先级考虑：

**局外系统完善**：

- 绝境战池扩容（首批 3-5 个）
- 定轨券多 sink：深层迷宫入场费 / 绝境战重置等
- 抽卡经济优化：辅助金币源（成就奖励、首击 Boss 加成等）以补足通关结算的 dopamine 频率

**深度难度**：

- 深层迷宫系统（难度 +1 / +2 + 开局强制负面策略卡）
- 决心上限的局外养成（永久扩容）
- 深层绝境战梯度（高层固定禁甲胄）

**留存机制**：

- 成就系统
- 每日挑战 / 种子赛
- 英雄传记 / lore 解锁
- **弱社交**：最佳通关时间排行、最高 DPS 排行等非强竞争性数据展示
- **游玩历史**：查看过去几次推塔记录——职业 / 装备 / 魔晶石 / 结束位置 / 最终 Boss 战 DPS 等。可为弱社交/成就系统提供数据源

**内容扩容**：

- 多 Boss 池（多层塔的层 Boss）
- 精英 / 小怪 / 事件素材池扩容
- 新进阶 job 和对应武器

> 施工顺序以 MVP 成功验证（玩法循环 fun + 6-9 小时不厌倦）为前提——任何 post-MVP 项目在 MVP 不 fun 的前提下都无意义。
